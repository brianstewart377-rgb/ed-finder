# ED:Finder Deployment Commands Reference

```bash
cd ~ && tar -xzf ed-finder-FINAL-COMPLETE.tar.gz && cd ed-finder-FINAL-COMPLETE && ./ULTIMATE_FIX.sh
```

## Manual Step-by-Step

### 1. Navigate and Extract
```bash
cd ~
tar -xzf ed-finder-FINAL-COMPLETE.tar.gz
cd ed-finder-FINAL-COMPLETE
```

### 2. Fix FastAPI Import
```bash
sed -i 's/from fastapi.middleware.base import BaseHTTPMiddleware/from starlette.middleware.base import BaseHTTPMiddleware/g' backend/main.py
```

### 3. Clean Old Containers
```bash
docker ps -a | grep ed-finder | awk '{print $1}' | xargs -r docker rm -f
```

### 4. Build Containers
```bash
docker compose build --no-cache
```

### 5. Start Services
```bash
docker compose up -d
```

### 6. Verify
```bash
docker compose ps
curl http://localhost/api/status
```

## Maintenance Commands

### View Logs
```bash
docker compose logs api      # API logs only
docker compose logs web      # Nginx logs only
docker compose logs -f       # Follow all logs (live)
docker compose logs --tail 50 api  # Last 50 lines
```

### Container Management
```bash
docker compose ps            # Check status
docker compose restart       # Restart all services
docker compose restart api   # Restart API only
docker compose restart web   # Restart Nginx only
docker compose stop          # Stop without removing
docker compose start         # Start stopped containers
docker compose down          # Stop and remove
docker compose up -d         # Start in background
```

### Rebuilding
```bash
# Rebuild everything
docker compose down
docker compose build --no-cache
docker compose up -d

# Rebuild API only
docker compose build --no-cache api
docker compose up -d

# Force recreate containers
docker compose up -d --force-recreate
```

### Troubleshooting
```bash
# Check container health
docker inspect ed-finder-api | grep -A 10 Health
docker inspect ed-finder-web | grep State

# Execute commands inside containers
docker compose exec api python3 --version
docker compose exec api python3 -c "import fastapi; print(fastapi.__version__)"
docker compose exec web nginx -t

# Check API directly
curl http://localhost:8000/api/health  # API container
curl http://localhost/api/health       # Through Nginx

# Check resource usage
docker stats ed-finder-api ed-finder-web
```

### Port Management
```bash
# Check what's using port 80
sudo lsof -i :80
sudo netstat -tulpn | grep :80

# Stop InfluxDB if it's blocking
sudo systemctl stop influxd
sudo systemctl disable influxd
```

### Cleanup
```bash
# Remove old images
docker image prune -a

# Remove unused volumes
docker volume prune

# Nuclear option (careful!)
docker system prune -a --volumes
```

## Emergency Recovery

### If everything is broken:
```bash
# Stop everything
docker compose down
docker ps -a | grep ed-finder | awk '{print $1}' | xargs -r docker rm -f

# Clean slate
docker image ls | grep ed-finder | awk '{print $3}' | xargs -r docker rmi -f

# Fresh start
cd ~/ed-finder-FINAL-COMPLETE
./ULTIMATE_FIX.sh
```

### If ULTIMATE_FIX.sh has issues:
```bash
# Make it executable
chmod +x ULTIMATE_FIX.sh

# Run with bash explicitly
bash ULTIMATE_FIX.sh

# Or run commands manually (see Manual Step-by-Step above)
```

## Backup Commands

### Create Backup
```bash
# Backup entire deployment
cd ~
tar -czf ed-finder-backup-$(date +%Y%m%d-%H%M%S).tar.gz ed-finder-FINAL-COMPLETE/

# Backup data only
tar -czf ed-finder-data-$(date +%Y%m%d-%H%M%S).tar.gz ed-finder-FINAL-COMPLETE/data/
```

### Restore Backup
```bash
# Stop current deployment
cd ~/ed-finder-FINAL-COMPLETE
docker compose down

# Extract backup
cd ~
tar -xzf ed-finder-backup-YYYYMMDD-HHMMSS.tar.gz

# Start restored version
cd ed-finder-FINAL-COMPLETE
docker compose up -d
```
