## What This Means

The compact summary proves the reconciliation artifact is reviewable without
loading the 2.1G source artifact into git or chat. It does not prove that every
station candidate is safe for a station-type production dry-run.

The broad station candidate set mixes several categories:

- confirmed/no-change evidence,
- candidate station updates,
- source-only missing-canonical evidence,
- ambiguous canonical matches,
- volatile evidence,
- station/body association blockers.

Stage 18J-P must therefore be narrowed to station-type-only candidates with
strong external identity proof and conservative risk exclusions.

## Stage 18J-P Readiness Verdict

Verdict: `Ready only with strict filter`.

The compact summary is valid and useful, but Stage 18J-P should not run against
all `298177` station candidates. A dry-run retry is appropriate only after the
station-type dry-run command is implemented or reconfirmed to apply the strict
filter below and to emit a bounded, reviewable dry-run artifact.

This verdict is not an approval record. It does not approve the compact summary
as a production artifact, does not approve a production dry-run, and does not
authorize canonical apply.

## Required Dry-Run Filter

Stage 18J-P station-type dry-run retry must require all of the following:

- explicit external identity match:
  - `source.market_id` matches `canonical.market_id`, or
  - `source.edsm_station_id` matches `canonical.edsm_station_id`;
- no internal primary-key-only identity proof;
- ambiguous canonical matches excluded;
- source-only insert candidates excluded;
- no station/body association writes;
- missing `station_body_name` must not block station-type comparison if station
  identity is otherwise externally proven, but it must remain a blocker for
  station/body-link work;
- volatile evidence excluded from station-type update candidates;
- fleet carriers, transient stations, and mobile station types excluded or
  labelled non-slot and blocked from canonical station-type apply;
- station-type field only;
- dry-run only;
- no canonical apply;
- explicit max-row bound required;
- reconciliation artifact checksum recorded in the dry-run output;
- output must be reviewable and compact;
- no approval record;
- no production artifact approval.

## Blockers / Exclusions

Exclude the following from Stage 18J-P dry-run input:

- `ambiguous_match`,
- `candidate_insert_missing_canonical`,
- candidates lacking external station identity proof,
- candidates where identity proof is only an internal primary key,
- candidates with volatile-only evidence,
- fleet carrier/transient/mobile station type candidates,
- any station/body association candidate,
- any candidate that would imply station/body-link writes,
- any candidate outside the explicit max-row bound.

If the filter cannot be proven before execution, Stage 18J-P remains blocked.

## Recommended Stage 18J-P Retry Scope

Recommended scope:

- station-type production dry-run only,
- exact externally proven station identity only,
- non-ambiguous canonical match only,
- non-volatile station-type evidence only,
- known permanent station types only,
- bounded row count set by an explicit operator-approved max row limit,
- compact output suitable for a Stage 18J-P2 dry-run review packet,
- no apply and no approval packet in the same step.

The first retry should be intentionally smaller than the full candidate set.
The `15014` high-confidence count is a review signal, not an automatic limit or
approval. The actual dry-run count must come from the strict filter and the
operator-approved max-row bound.

## Final Recommendation

Do not run Stage 18J-P against all `298177` station candidates. Proceed only
with a strict, externally identified, non-ambiguous, non-volatile,
station-type-only dry-run filter, an explicit max-row bound, and compact
reviewable output. Keep station/body association work blocked, keep canonical
apply unauthorized, and keep Stage 18J-P blocked until a separate prompt
implements or reconfirms the filtered dry-run scope.

Stage 18J-P-filter is the implementation hardening step for that filter. It
does not run the production dry-run or approve any artifact. Stage
18J-P-dryrun-ops adds the operator wrapper needed before that future dry-run.
