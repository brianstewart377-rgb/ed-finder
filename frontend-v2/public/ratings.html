Scoring Model Notes

ED Finder now treats development score and archetype assessment as the primary operator-facing ranking model. Legacy ratings still appear in some places as compatibility context while the remaining backend contracts finish moving over.

Current rule: if you are looking at Finder, Compare, Map selection, Pinned, Watchlist, or System Detail in the v2 app, the main score you see is intended to be the archetype-led development score. The old v3.4 rating model is no longer the lead signal.

What The Development Score Means

The development score is the archetype engine's top-line view of how promising a system is for colony development. It is tied to the system's strongest colony archetype, not just a legacy economy-pair rating. In practice, that means the app is trying to answer:

  • What colony role does this system fit best?
  • How strong is that fit?
  • How buildable is the system once slot and body constraints are considered?
  • How clean or contaminated is the archetype profile?

Why Legacy Ratings Still Exist

Parts of the backend and generated OpenAPI contract still carry v3.4 naming such as _rating, sort_by=rating, or min_rating. Those paths are being retired in place rather than ripped out in one break, so the app can keep working while the contracts migrate.

Compatibility policy

  • New UI surfaces should lead with development score and archetype language.
  • Old rating fields may still be returned and displayed, but only as compatibility context.
  • Legacy API names are accepted on input where needed so older callers do not immediately break.

Legacy v3.4 Rating Model

The old rating model was based on build_ratings.py and a strongest economy-pair approach. It remains useful as historical context, calibration reference, and fallback data for older saved entries, but it is no longer the intended end-state scoring model for v2.

Legacy reference scope: economy-pair strength, slot heuristics, rarity gates, and standout-body bonuses. This was valuable groundwork, but the archetype engine is the path the app is actively moving toward.

Mechanics Reference

The colony mechanics background still comes from the same Elite Dangerous colonisation research sources, especially the Mega-Guide and the redesign docs in this repository. What changed is the app contract: those mechanics are increasingly expressed as archetypes and development potential rather than a single legacy rating number.

Active app path: archetype-led development scoring in v2.
